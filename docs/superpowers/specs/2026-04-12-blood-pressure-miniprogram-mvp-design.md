# 家庭血压心率监测小程序 MVP 设计

日期：2026-04-12

## 目标

先交付一个家庭内部可用的微信小程序 MVP，让家人可以完成“创建家庭组 → 邀请成员 → 录入血压心率 → 查看趋势统计 → 导出图表/报告”的闭环。

V1 不做公开上架优化、PDF、真实订阅消息推送、用药提醒或完整药物管理。药物管理在家庭页保留占位入口。

V1 不提前暴露血糖入口，也不实现血糖数据模型、页面、图表或报告。为 V2 保留产品概念上的扩展空间：家庭组命名和邀请文案使用“健康记录”，但页面、数据模型和导航仍只实现血压心率记录。后续血糖可作为同一家庭组下的模块切换加入。

V2 可增加用户级默认模块偏好，例如 `users.preferences.defaultModule = 'bp' | 'glucose'`，用于决定打开小程序后默认进入血压心率或血糖模块。V1 不显示“设为主页面”入口，也不实现该字段；启动默认进入血压心率数据页。

## 已确认决策

- 路线：功能闭环优先，UI 按 `mockup.html` 精简落地。
- UI：默认偏老人友好，不做两套布局；设置中保留“标准 / 大 / 超大”字号微调。
- 图表：自写 Canvas，不使用 `wx-charts`。
- 报告：生成结构化图片报告，保存到相册。
- 单独图表导出：血压统计和心率统计都支持按当前 7/30/90 天周期单独下载图表图片。
- 邀请：主流程使用微信分享卡片进入加入确认页；邀请码只做备用。
- 权限：默认成员只读，管理员可授权成员录入和编辑。
- 通知：V1 保存通知对象和阈值配置，不接真实微信订阅消息推送。

## 页面结构

### 数据页 `pages/data`

默认首页。展示最近一次记录、添加记录按钮、7/30/90 天周期切换、血压统计、心率统计、图表和入口。

血压统计模块包含：

- 参考范围内/需关注次数和均值。
- Canvas 血压趋势图。
- “下载图表”按钮，导出当前周期血压趋势图片。
- `?` 说明入口，弹出“血压数据须知”。

心率统计模块包含：

- 参考范围内/需关注次数和均值。
- Canvas 心率柱状图。
- “下载图表”按钮，导出当前周期心率趋势图片。
- `?` 说明入口，弹出“心率数据须知”。

数据页还提供“全部记录”入口。

### 家庭页 `pages/family`

展示被监护人档案、家庭成员、邀请入口、权限管理入口、报告入口和设置入口。

家庭页保留功能入口 emoji：

- `📄 生成就诊报告`
- `💊 药物管理`

药物管理入口在 V1 显示“即将上线”，不进入实际功能。

### 加入家庭页 `pages/join-family`

分享卡片打开的确认页。展示邀请来源和家庭名称，用户点击“同意加入”后调用 `joinFamily`。

失败状态需要覆盖：

- 邀请 token 无效或过期。
- 已经是家庭成员。
- 家庭成员已达 10 人上限。
- 登录失败或网络错误。

### 添加/编辑记录页 `pages/add-record`

字段：

- 高压。
- 低压。
- 心率。
- 测量时间。
- 晨/晚测；仅在设置打开时显示。

不做备注字段。

### 全部记录页 `pages/records`

按日期分组显示全部记录，支持展开/收起、编辑、删除。提供完整就诊报告导出入口。

### 设置页 `pages/settings`

分组：

- 通知设置：阈值、通知对象配置；V1 只保存配置。
- 图表设置：晨/晚测标注、早晚分线。
- 显示设置：字体大小。
- 成员权限：`可录入`、`可编辑`。
- 关于。

## 数据模型

### `users`

保存登录用户身份：

```text
_id        openid
nickname   昵称
avatarUrl  头像
familyId   所属家庭组
role       admin | member | ''
createdAt
```

### `families`

保存家庭组信息：

```text
_id
displayName    家庭显示名，默认“家庭健康记录”
inviteCode     备用邀请码
inviteToken    分享邀请 token
createdBy      admin openid
members        成员数组
profile        被监护人档案
settings       家庭设置
createdAt
```

`members[]`：

```text
openid
role       admin | member
nickname
avatarUrl
canWrite   是否可录入
canEdit    是否可编辑/删除家庭记录
joinedAt
```

`profile` 默认值：

```text
name
birthYear
targetSystolic    默认 135
targetDiastolic   默认 85
targetHRMin       默认 60
targetHRMax       默认 80
medications
emergencyContact
```

`settings`：

```text
alertSystolic
alertDiastolic
notifyAll
notifyMemberIds
morningEveningLabel
splitLines
fontSize
```

### `records`

```text
_id
familyId
systolic
diastolic
heartRate
measuredAt
period       morning | evening | null
recordedBy
createdAt
updatedAt
```

## 权限模型

- admin：可查看、录入、编辑、删除、改档案、改设置、授权成员。
- member 默认：只读。
- member 且 `canWrite=true`：可录入。
- member 且 `canEdit=true`：可编辑/删除家庭记录。

客户端根据权限隐藏按钮。云函数必须做最终权限校验，不能只依赖前端。

## 邀请流程

主流程：

1. 管理员在家庭页点击“邀请家人”。
2. 小程序通过分享卡片发送邀请。
3. 分享 path 携带 `inviteToken`，打开 `pages/join-family/join-family?inviteToken=...`。
4. 加入页展示邀请信息。邀请页使用 `families.displayName`，默认“家庭健康记录”。如果用户填写了被监护人档案姓名，可以建议显示名为“{档案姓名}的健康记录”，但系统不默认“爸爸”“妈妈”等亲属称呼。
5. 用户点击“同意加入”。
6. `joinFamily` 校验 token、人数上限和重复加入，然后写入 `families.members[]` 与 `users.familyId/role`。

备用流程：

- 展示或复制 `inviteCode`。
- 已打开小程序的家人可以手动输入邀请码加入。

家庭页入口规则：

- 未加入家庭组时，家庭页空状态显示“创建家庭组”和“加入家人的家庭组”两个入口。
- 点击“加入家人的家庭组”后展示输入邀请码弹窗或同页表单，包含 6 位邀请码输入框、加入按钮和提示文案：“如果家人发给你的是微信邀请卡片，直接点开卡片即可加入。”
- 已加入家庭组时，家庭成员区域显示“邀请”入口。点击后展示操作面板：“发送微信邀请”“复制邀请码”“取消”。
- 微信邀请卡片和加入确认页文案使用通用健康记录命名，例如“Ella 邀请你加入「家庭健康记录」”。不写死血压记录，也不写死被监护人关系。

## 参考规则

这些规则只用于健康记录、图表统计和状态提示，不作为诊断、治疗或用药依据。档案中的目标值可以按医生建议调整。

### 血压

默认按家庭自测场景使用 `<135/85 mmHg` 作为参考目标。

| 家庭自测血压 | UI 状态 |
|---|---|
| `<135` 且 `<85` | 参考范围内 |
| `135-159` 或 `85-99` | 偏高，建议关注 |
| `160-179` 或 `100-109` | 明显偏高 |
| `>=180` 或 `>=110` | 很高 |
| `<90` 或 `<60` | 偏低，建议关注 |

### 心率

默认面向高血压患者家庭监测，使用 `60-80 bpm` 作为参考范围。

| 静息心率 | UI 状态 |
|---|---|
| `60-80 bpm` | 参考范围内 |
| `81-100 bpm` | 偏快，建议关注 |
| `>100 bpm` | 明显偏快 |
| `50-59 bpm` | 偏慢，建议关注 |
| `<50 bpm` | 明显偏慢 |

### 须知弹窗

血压数据须知：

> 血压参考范围和状态提示参考《中国高血压防治指南（2024年修订版）》及家庭血压管理常用标准。本小程序的状态提示和图表结果仅供健康记录与就诊沟通参考，不作为诊断、治疗或用药依据。个体情况存在差异，请以医生诊疗结果及医嘱为准。

心率数据须知：

> 心率参考范围和状态提示参考《中国高血压患者心率管理多学科专家共识（2021年版）》及常用静息心率范围。本小程序的状态提示和图表结果仅供健康记录与就诊沟通参考，不作为诊断、治疗或用药依据。个体情况存在差异，请以医生诊疗结果及医嘱为准。

## 图表和导出

图表数据统一由 `utils/chart.js` 生成，页面展示和导出图片使用同一份聚合结果。

默认聚合：

- 7/30/90 天。
- 一天多条记录时使用日均值。
- 晨/晚测分线打开后，血压图输出 morning/evening 两组数据。

导出：

- 血压模块导出当前周期血压图图片。
- 心率模块导出当前周期心率图图片。
- 就诊报告导出结构化图片，包含档案、周期统计、趋势小图和最近记录。

## 云函数

V1 需要：

- `login`
- `createFamily`
- `joinFamily`
- `getFamily`
- `getRecords`
- `saveRecord`
- `deleteRecord`
- `saveSettings`
- `updateMemberPermission`

所有写操作云函数都要校验调用者是否属于该家庭，以及是否具备对应权限。

## UI 约定

- 默认老人友好：大字号、大按钮、少层级。
- 状态不只靠颜色，还显示文字。
- 家庭页功能 emoji 保留。
- 首页/统计模块图标优先使用统一图标资源，`?` 说明入口保持轻量。
- “下载图表”是标题右侧轻量按钮，不抢主要内容层级。
- 药物管理在 V1 只显示占位，不进入未完成页面。

## 验证策略

开发完成后至少验证：

- 无家庭组时能引导创建或加入。
- 分享邀请 token 能加入家庭组。
- 邀请 token 无效、重复加入、成员满员都有提示。
- 只读成员看不到录入/编辑入口，云函数也拒绝越权。
- `canWrite` 成员可录入。
- `canEdit` 成员可编辑/删除家庭记录。
- 7/30/90 天图表展示和下载使用一致数据。
- 血压/心率参考规则、统计和须知弹窗一致。
- 大/超大字号下主要文本不溢出。
- 图表导出和报告导出能保存到相册，并能处理相册权限失败。
