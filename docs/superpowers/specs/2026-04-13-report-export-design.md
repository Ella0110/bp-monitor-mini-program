# 就诊报告导出设计

日期：2026-04-13

## 目标

让家属可以从家庭页生成一张适合就诊沟通使用的血压心率报告图片。报告先做功能闭环和内容完整性，不做最终 UI 精修，不做 PDF，不做微信分享卡片。

## 范围

本阶段包含：

- 家庭页 `📄 生成就诊报告` 入口进入报告预览页。
- 新增报告预览页 `pages/report/report`。
- 报告预览页支持 `7天 / 30天 / 90天` 周期切换。
- 报告预览页展示摘要、趋势图、最近记录和免责声明。
- 保存报告图片到相册。

本阶段不包含：

- 全部记录页入口。
- PDF 导出。
- 微信分享卡片。
- 最终 UI 精修。
- 血糖报告。

## 入口与流程

1. 用户在家庭页点击 `📄 生成就诊报告`。
2. 小程序进入 `pages/report/report`。
3. 页面默认选择 `30天`。
4. 用户可切换 `7天 / 30天 / 90天`。
5. 页面按当前周期加载家庭记录并刷新报告预览。
6. 用户点击 `保存报告图片`。
7. 小程序生成报告图片并保存到相册。

如果当前用户没有加入家庭组，家庭页不显示报告入口；如果报告页无法获取 `familyId`，显示“请先创建或加入家庭组”。

## 报告预览内容

报告预览按以下顺序展示：

1. 标题：`血压心率就诊报告`。
2. 基本信息：
   - 家庭健康记录名，使用 `families.displayName`，默认 `家庭健康记录`。
   - 被监护人姓名，使用 `families.profile.name`，为空时显示 `未设置`。
   - 报告周期：`近7天 / 近30天 / 近90天`。
   - 生成时间。
3. 摘要：
   - 记录次数。
   - 血压均值。
   - 心率均值。
   - 血压需关注次数。
   - 心率需关注次数。
4. 血压趋势图：
   - 复用当前 Canvas 图表逻辑。
   - 高压/低压两条线。
   - 135/85 参考线。
   - 异常记录点红色标记。
5. 心率趋势图：
   - 复用当前 Canvas 图表逻辑。
   - 心率柱状图。
   - 60/80 参考线。
   - 异常柱红色展示。
6. 最近记录：
   - 最多显示最近 10 条。
   - 每条包含测量时间、高压/低压、心率、血压状态和心率状态。
7. 免责声明：
   - `本报告仅供健康记录与就诊沟通参考，不作为诊断、治疗或用药依据。个体情况存在差异，请以医生诊疗结果及医嘱为准。`

## 数据来源

报告页不新增云函数。

- 家庭信息：优先复用 `getFamily` 云函数。
- 记录数据：复用 `getRecords` 云函数，传入当前家庭 `familyId` 和周期起始时间 `since`。
- 统计规则：复用 `utils/health-rules.js`。
- 图表数据：复用 `utils/chart-data.js`。
- 图表绘制：复用 `utils/canvas-charts.js`。

## 图片生成

报告图片使用隐藏 Canvas 生成。

图片内容和预览内容一致，但布局可以为图片导出做适度压缩：

- 宽度使用固定导出宽度。
- 高度根据最近记录条数计算。
- 顶部保留标题和基本信息。
- 中间绘制摘要、血压图和心率图。
- 底部绘制最近记录和免责声明。

保存逻辑：

1. 绘制隐藏 Canvas。
2. 调用 Canvas 临时文件接口生成图片。
3. 调用保存到相册接口。
4. 成功提示 `已保存到相册`。
5. 没有记录时仍允许保存报告，但摘要显示 0 次，图表区域显示 `当前周期暂无记录`。
6. 如果用户未授权相册权限，引导打开设置。
7. 如果图片生成失败，提示 `报告生成失败`。

## 错误处理

- 家庭组不存在：显示 `请先创建或加入家庭组`。
- 加载失败：显示 toast `报告加载失败`，保留页面可重试。
- 保存失败：显示 toast `保存失败`。
- 相册权限拒绝：弹窗说明并提供 `去设置`。
