# 血压心率图表与下载设计

日期：2026-04-12

## 目标

在数据页补齐当前 7/30/90 天周期的图表展示和单独下载能力。先做功能闭环，不做最终 UI 精修，也不做完整就诊报告导出。

## 范围

本阶段包含：

- 血压统计区展示 Canvas 折线图。
- 心率统计区展示 Canvas 柱状图。
- 血压统计区提供“下载图表”按钮，保存当前周期血压图到相册。
- 心率统计区提供“下载图表”按钮，保存当前周期心率图到相册。

本阶段不包含：

- 完整就诊报告图片。
- PDF。
- UI 最终视觉精修。
- 血糖图表。
- 真实订阅消息。

## 图表方案

### 血压图

血压图使用自写 Canvas。

- 横轴：当前周期内有记录的日期，按测量时间从早到晚排列；7/30/90 天都复用同一绘制逻辑。
- 纵轴：血压数值，自动根据数据和参考线取合适范围，至少覆盖 85 和 135。
- 折线：高压一条线，低压一条线。
- 参考线：显示 135 和 85 两条虚线，标注为参考线 135/85。
- 图例：高压、低压、参考线 135/85。
- 空状态：没有记录时显示“当前周期暂无记录”，不绘制空图。

### 心率图

心率图使用自写 Canvas。

- 横轴：当前周期内有记录的日期，按测量时间从早到晚排列。
- 纵轴：心率数值，自动根据数据和参考范围取合适范围，至少覆盖 60 和 80。
- 柱状：每条记录一根柱，心率超过 80 或低于 60 时使用关注色。
- 参考线：显示 60 和 80 两条虚线，标注为参考范围 60-80。
- 图例：心率、参考范围 60-80。
- 空状态：没有记录时显示“当前周期暂无记录”。

## 下载行为

点击“下载图表”时：

1. 使用当前页面数据和当前周期重新绘制到导出 Canvas。
2. 调用小程序 Canvas 临时文件接口生成图片。
3. 调用保存到相册接口。
4. 成功后提示“已保存到相册”。
5. 如果用户未授权相册权限，引导用户打开设置。
6. 如果当前周期没有记录，提示“当前周期暂无可下载图表”。

导出的图片内容包含：

- 标题：血压趋势图或心率趋势图。
- 周期：7天、30天或90天。
- 图表主体。
- 图例。
- 简短提示：仅供健康记录与就诊沟通参考。

## 数据与权限

图表只使用 `pages/data` 已加载的 `records`，不新增云函数。周期切换仍调用现有 `getRecords`。

下载图片在本地完成，不写入云数据库，也不上传云存储。

## 实现边界

图表绘制逻辑放到独立工具模块，避免把 `pages/data/data.js` 变得过长。数据页只负责：

- 拉取记录。
- 触发绘制。
- 触发下载。
- 处理空状态和 toast。

工具模块负责：

- 计算坐标范围。
- 绘制坐标网格、参考线、折线、柱状、图例。
- 生成导出图片。
